# Transform to Talent Marketplace - Production Deployment Guide\n\n## 🚀 Production Deployment\n\n### Prerequisites\n\n- Docker & Docker Compose\n- PostgreSQL database\n- Domain name with SSL certificate\n- M-Pesa production credentials\n- Sentry account (for error tracking)\n- Redis instance (Upstash recommended)\n\n### Quick Start\n\n1. **Clone and Setup**\n   ```bash\n   git clone <repository-url>\n   cd Transform-to-Talent-Marketplace\n   ```\n\n2. **Configure Environment**\n   ```bash\n   cp .env.production.example .env.production\n   # Edit .env.production with your production values\n   ```\n\n3. **Deploy**\n   ```bash\n   chmod +x deploy.sh\n   ./deploy.sh\n   ```\n\n### Environment Configuration\n\n#### Required Variables\n\n```env\n# Database\nDATABASE_URL=\"postgresql://username:password@postgres:5432/talent_marketplace\"\n\n# Authentication\nNEXTAUTH_URL=https://yourdomain.com\nNEXTAUTH_SECRET=your_32_character_secret_here\n\n# M-Pesa Production\nMPESA_CONSUMER_KEY=your_production_key\nMPESA_CONSUMER_SECRET=your_production_secret\nMPESA_SHORTCODE=your_production_shortcode\nMPESA_PASSKEY=your_production_passkey\n```\n\n#### Optional Services\n\n```env\n# Error Tracking\nSENTRY_DSN=your_sentry_dsn\n\n# Redis Cache\nUPSTASH_REDIS_REST_URL=your_redis_url\nUPSTASH_REDIS_REST_TOKEN=your_redis_token\n\n# File Upload\nUPLOADTHING_SECRET=your_uploadthing_secret\nUPLOADTHING_APP_ID=your_uploadthing_app_id\n```\n\n### Production Features\n\n✅ **Security**\n- Rate limiting\n- CSRF protection\n- SQL injection prevention\n- Security headers\n- Input sanitization\n\n✅ **Performance**\n- Docker containerization\n- Database connection pooling\n- Redis caching\n- Image optimization\n- Gzip compression\n\n✅ **Monitoring**\n- Health checks\n- Error tracking (Sentry)\n- Performance logging\n- Database metrics\n- Request logging\n\n✅ **Reliability**\n- Automated backups\n- Graceful shutdowns\n- Database migrations\n- Container health checks\n\n### SSL Configuration\n\n1. **Obtain SSL Certificate**\n   ```bash\n   # Using Let's Encrypt\n   certbot certonly --standalone -d yourdomain.com\n   ```\n\n2. **Copy Certificates**\n   ```bash\n   mkdir ssl\n   cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem ssl/cert.pem\n   cp /etc/letsencrypt/live/yourdomain.com/privkey.pem ssl/key.pem\n   ```\n\n### Database Setup\n\n1. **Run Migrations**\n   ```bash\n   docker-compose exec app npx prisma migrate deploy\n   ```\n\n2. **Seed Database**\n   ```bash\n   docker-compose exec app npm run db:seed\n   ```\n\n### Monitoring Setup\n\n1. **Start Monitoring Stack**\n   ```bash\n   docker-compose -f docker-compose.monitoring.yml up -d\n   ```\n\n2. **Access Dashboards**\n   - Grafana: http://localhost:3001 (admin/admin123)\n   - Prometheus: http://localhost:9090\n\n### Backup & Recovery\n\n1. **Automated Backups**\n   ```bash\n   # Setup daily backups\n   chmod +x backup.sh\n   crontab -e\n   # Add: 0 2 * * * /path/to/backup.sh\n   ```\n\n2. **Manual Backup**\n   ```bash\n   ./backup.sh\n   ```\n\n3. **Restore from Backup**\n   ```bash\n   gunzip backups/backup_YYYYMMDD_HHMMSS.sql.gz\n   docker-compose exec -T postgres psql -U postgres -d talent_marketplace < backups/backup_YYYYMMDD_HHMMSS.sql\n   ```\n\n### Health Checks\n\n- **Application**: `GET /api/health`\n- **Database**: Automatic connection testing\n- **Redis**: Ping test (if configured)\n\n### Scaling\n\n1. **Horizontal Scaling**\n   ```yaml\n   # In docker-compose.yml\n   app:\n     deploy:\n       replicas: 3\n   ```\n\n2. **Load Balancer**\n   - Nginx handles load balancing\n   - Configure upstream servers\n\n### Security Checklist\n\n- [ ] SSL certificates installed\n- [ ] Environment variables secured\n- [ ] Database credentials rotated\n- [ ] Rate limiting configured\n- [ ] Security headers enabled\n- [ ] Backup encryption enabled\n- [ ] Monitoring alerts configured\n\n### Troubleshooting\n\n1. **Check Logs**\n   ```bash\n   docker-compose logs app\n   docker-compose logs postgres\n   ```\n\n2. **Health Check**\n   ```bash\n   curl -f http://localhost:3000/api/health\n   ```\n\n3. **Database Connection**\n   ```bash\n   docker-compose exec postgres psql -U postgres -d talent_marketplace\n   ```\n\n### Performance Optimization\n\n1. **Database Indexing**\n   - Review slow query logs\n   - Add indexes for frequent queries\n\n2. **Caching Strategy**\n   - Redis for session storage\n   - CDN for static assets\n\n3. **Resource Monitoring**\n   - CPU and memory usage\n   - Database connection pool\n   - Response times\n\n### Support\n\nFor production issues:\n1. Check application logs\n2. Review monitoring dashboards\n3. Verify environment configuration\n4. Test database connectivity\n\n---\n\n## Development\n\n```bash\nnpm install\nnpm run dev\n```\n\nVisit http://localhost:3000\n\n## License\n\nMIT License
